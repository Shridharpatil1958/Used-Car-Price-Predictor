# Used Car Price Predictor

Predicts the fair resale price of a used car (Indian market, CarDekho dataset) using a tuned
XGBoost regression model, with a Streamlit UI and SHAP-based explanations for every prediction.

## Results

| Model | CV RMSE (log) | Test RMSE (₹) | Test MAE (₹) | Test R² |
|---|---|---|---|---|
| Linear Regression (baseline) | 0.2033 | 211,968 | 110,499 | 0.884 |
| Random Forest | 0.1728 | 157,455 | 84,811 | 0.936 |
| **XGBoost (tuned)** | **0.1638** | **145,470** | **80,328** | **0.945** |

The tuned XGBoost model is used in production. On average, predictions are within ~₹80,000 of
the actual selling price, explaining ~94.5% of price variance.


## Usage

**1. Run EDA** (optional — figures are already generated in `reports/figures/`):
```bash
python src/eda.py
```

**2. Train the model** (regenerates everything in `models/`):
```bash
python src/train.py
```

**3. Generate SHAP explainability plots:**
```bash
python src/explain.py
```

**4. Run the web app:**
```bash
streamlit run app/streamlit_app.py
```
Then open the URL Streamlit prints (usually `http://localhost:8501`).

**5. Predict programmatically:**
```python
from src.predict import predict_price

result = predict_price({
    "brand": "Maruti", "model": "Swift", "vehicle_age": 4,
    "km_driven": 35000, "seller_type": "Dealer", "fuel_type": "Petrol",
    "transmission_type": "Manual", "mileage": 20.5, "engine": 1197,
    "max_power": 85.0, "seats": 5,
})
print(result)
# {'predicted_price': 502675, 'low_estimate': 426940, 'high_estimate': 591844}
```

## Key Design Decisions

- **Target encoding for `brand`/`model`**: too high-cardinality for one-hot encoding; mean
  log-price per category is computed on the training split only, to avoid leakage.
- **Log1p transform on `selling_price`**: raw prices are heavily right-skewed (skew ≈ 10);
  modeling in log space stabilizes variance and improves fit, especially for the linear baseline.
- **Outlier handling**: rows with `km_driven > 300,000`, `seats == 0`, `max_power <= 0`, or price
  above the 99.5th percentile are dropped from training (109 rows) as likely data errors or
  extreme luxury outliers that would distort the model. At inference time, inputs are clipped
  rather than rejected, so the app never crashes on an edge-case input.
- **Price range, not just a point estimate**: the app reports a ±1 std-dev range (in log space,
  converted back) around the prediction, based on residuals on the held-out test set.
- **SHAP TreeExplainer**: fast, exact explanations for tree-based models; used both for global
  feature importance (`explain.py`) and per-prediction waterfall plots in the app.

## Data Quality Notes

- The raw CSV had a stray index column (`Unnamed: 0`) that masked 167 true duplicate rows —
  these are dropped during cleaning.
- No missing values in this dataset (already cleaner than typical scraped listings).
